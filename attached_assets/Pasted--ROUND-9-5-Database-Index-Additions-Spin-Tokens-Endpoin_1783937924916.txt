# ROUND 9.5 — Database Index Additions + Spin-Tokens Endpoint Cap

## Context

This is a narrow, low-risk performance round. The database schema currently has almost no explicit indexes: the only ones that exist are `IDX_session_expire` on `sessions` and the two indexes on `support_messages`. Postgres does NOT auto-index foreign key columns, so every dashboard/analytics query filtering by `store_id` + timestamp is doing sequential scans, and every cascade delete scans child tables. Additionally, `GET /api/spin-tokens` returns every token ever created for a store with no limit.

This round adds indexes to `shared/schema.ts` and caps that one endpoint. **Nothing else changes.**

## Scope — exactly two changes

### Change 1: Add indexes in `shared/schema.ts`

Add index definitions using Drizzle's `index()` helper (already imported in the file) via each table's third-argument callback, following the exact same style already used on `supportMessages`:

```ts
}, (table) => ({
  someIdx: index("idx_name").on(table.col1, table.col2),
}));
```

For tables that already have a callback (e.g. `loyaltyCards` with `uniqueUserCustomer`), ADD the index entries to the existing callback object — do not remove or rename the existing unique constraints.

Add exactly these indexes and nothing more:

| Table | Index name | Columns |
|---|---|---|
| `loyaltyTransactions` | `idx_ltx_store_created` | `storeId, createdAt` |
| `loyaltyTransactions` | `idx_ltx_card` | `loyaltyCardId` |
| `spins` | `idx_spins_store_spun` | `storeId, spunAt` |
| `spins` | `idx_spins_customer` | `customerId` |
| `spins` | `idx_spins_token` | `tokenId` |
| `customers` | `idx_customers_store_created` | `storeId, createdAt` |
| `loyaltyCards` | `idx_cards_store` | `storeId` |
| `loyaltyCards` | `idx_cards_customer` | `customerId` |
| `spinTokens` | `idx_tokens_store_created` | `storeId, createdAt` |
| `shifts` | `idx_shifts_store_date` | `storeId, shiftDate` |
| `menuItems` | `idx_menu_items_category` | `categoryId` |
| `appleWalletDevices` | `idx_awd_customer` | `customerId` |

### Change 2: Cap `GET /api/spin-tokens` in `server/routes.ts`

The endpoint currently is:

```ts
const tokens = await db
  .select()
  .from(spinTokens)
  .where(eq(spinTokens.storeId, req.storeId!))
  .orderBy(desc(spinTokens.createdAt));
```

Add `.limit(100)` to the end of this query chain. That is the ONLY modification to this endpoint: same route path, same auth/permission middleware, same response shape (array of full token rows), same ordering. Do not add pagination, query params, filters on `isUsed`/`expiresAt`, or change the response structure.
